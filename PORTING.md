# Porting Map

## Codex Continuation Rule

When a bounded porting chunk is implemented and verified, do not stop at that milestone unless the user explicitly asks for a pause. Continue by selecting the next small Python-parity gap from this map, adding or tightening tests, implementing the change, running focused and then broad verification, and cleaning temporary build artifacts. Treat green tests as a checkpoint for the next chunk, not as the end of the task.

## Porting Priority

Prioritize parity work that affects the bundled/default CTB action route first, especially `fixtures/ctb_actions_input.txt`, the default browser sample, and helper flows used by that route. Once the default CTB route and its common helpers render cleanly and quickly with no important parity gaps, move on to less-common behavior needed by other action files, alternate routes, and broader tracker completeness.

## Recommended Validation Fixtures

Create fixtures from Python before replacing each layer:

- RNG initial values and first N rolls for several seeds
- Parser normalized line output for representative commands
- Encounter state after each encounter line
- Full CTB rendered output for the default seed
- Focused outputs for Tanker, Garuda, Chocobo Eater, Spherimorph, and multi-zone encounters

## Performance Target

The first benchmark target should be the Tanker custom action path:

- Python desktop/local target: about `0.9s`
- Pyodide baseline: about `2.0s`
- Rust/WASM target: below local Python once equivalent checkpointing is implemented
